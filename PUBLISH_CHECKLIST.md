# Publish Checklist

Run this before each push to `main`.

1. Confirm file has boundary framing where needed.
2. Confirm high-risk claims are tagged as observation/interpretation/hypothesis.
3. Remove any implied medical instruction or treatment wording.
4. Keep raw measurements unchanged when present.
5. Verify links resolve and use canonical URLs.
6. Ensure filename is clear and stable.
7. Confirm page fits intended lane: public, community, or private.
8. Check spelling/typos in titles and key sections.
9. Update `REVIEW_QUEUE.md` status after merge.
10. Commit with a clear message describing scope.
